Getting Started

What is Gitana SDK?

Gitana SDK is a set of examples and sample sites which help you get started with writing Gitana Cloud CMS client application. Gitana Cloud CMS is an infrastructure cloud service aimed at providing all the back-end capabilities that developers need to build powerful, next-generation mobile and web applications. It delivers the easiest and most cost-effective way for developers to incorporate scalable, enterprise-class content services into multi-device HTML5 applications.

Start by checking out hosted demos

To get a feel for what Gitana SDK is about, check out the hosted demos. In the demos, the left-side navigation lists basic feature examples under Gitana section, examples for using Gitana with jQuery Template engine under Gitana + jQuery Template section, examples for using Gitana with Alpaca forms engine under Gitana + Alpaca section and links to Social Graph site and sample Ratchet site under Sample Applications section. Each demo comes with a short description and allows you to view source code by clicking on "Source Code" button.

Download Gitana SDK

Check out download section of Gitana SDK front page for links to individual SDK components. You can also just download Gitana SDK examples zip, a self-contained file which includes everything you need to run Gitana SDK examples locally.

Intro to writing Gitana client application

Use your favorite text editor to create an empty html file. You will first need to include Gitana JavaScript library. You can either use local JS file you downloaded or just use hosted one. It is recommended to use minified version for production and uncompressed version for development. 

                <script type="text/javascript" src="http://code.cloudcms.com/gitana-javascript-driver/1.0.0-SNAPSHOT/gitana.min.js">
                </script>

Now include other Gitana SDK components only if they are needed.

Alpaca Library

You will need Alpaca for content rendition or form service. To use Alpaca, you will first need to include jQuery library (1.5 or later) and jQuery UI library (1.8.8 or later). jQuery UI library is not required for running Alpaca but highly recommended. 

                <script type="text/javascript" src="http://code.cloudcms.com/alpaca/js/alpaca-${project.version}.min.js">
                </script>
                <link type="text/css" href="http://code.cloudcms.com/alpaca/css/alpaca-${project.version}.min.css"
                rel="stylesheet"/>

Then include link to minified Alpaca JavaScript library and minified Alpaca stylesheet in its head section 

                <script type="text/javascript" src="http://code.cloudcms.com/alpaca/js/alpaca-${project.version}.min.js">
                </script>
                <link type="text/css" href="http://code.cloudcms.com/alpaca/css/alpaca-${project.version}.min.css"
                rel="stylesheet"/>

jQuery Template Engine

jQuery Template engine can be used for rendering Gitana content. You can either download it from its website. Or you can use the convenient wrapper JavaScript class that Gitana SDK provides for jQuery Template engine. 

                <script type="text/javascript" src="http://code.cloudcms.com/sdk/js/gitana-sdk-jquery-template-${project.version}.min.js">
                </script>

Ratchet MVC

To use Ratchet MVC for your application, you will first need to include Ratchet MVC JavaScript library. 

                <script type="text/javascript" src="http://code.cloudcms.com/ratchet/${project.version}/ratchet-${project.version}.min.js">
                </script>

And then include either jQuery Template Plugin

                <script type="text/javascript" src="http://code.cloudcms.com/ratchet/${project.version}/plugins/jQueryTemplateEngine.js">
                </script>

Or Alpaca Plugin

                <script type="text/javascript" src="http://code.cloudcms.com/ratchet/${project.version}/plugins/AlpacaEngine.js">
                </script>

Setting up proxy

Since Gitana SKD examples interact with Gitana Cloud CMS through REST services, it is required to run the examples either in a web server (such as Apache Web Server ) or a web application server (such as Tomcat or Jetty).

Because we are using Gitana JavaScript driver for SDK examples, we will have to set up a web proxy to forward cross-Domain XMLHttpRequest calls to endpoint "http://api.cloudcms.com:80".

For web application servers, you can follow the setup in Gitana SDK for instruction of using Gitana proxy servlet which you can download from Gitana maven repository. 

		    <!-- Gitana Maven repository -->
			<id>cloudcms-public</id>
			<name>cloudcms-public</name>
			<url>http://maven.cloudcms.com/public</url>

	        <!-- Gitana Proxy Servlet -->
            <groupId>org.gitana</groupId>
            <artifactId>gitana-servlets</artifactId>
            <version>1.0.0-SNAPSHOT</version>

Or you can also use other web proxy servlets such as Jetty Asynchronous Proxy Servlet.

Please note that if you are not running your webb application as root web app. You will need to initialize Gitana driver with serverURL parameter such as 

                var gitana = new Gitana({
                    "serverURL" :"/YOUR_WEBAPP_NAME/proxy"
                });

For web servers, you need to setup a similar reverse proxy that will forward requests to "http://api.cloudcms.com:80". As default, the requests will first be sent to endpoint "/proxy". If you don't want to use the default, you will then need to provide the serverURL parameter when you initialize Gitana driver such as 

                var gitana = new Gitana({
                    "serverURL" :"/YOUR_CUSTOM_PROXY_NAME"
                });

Here is a sample configuration for setting Apache proxy 

        ProxyRequests Off

        ProxyPass /proxy http://api.cloudcms.com:80
        ProxyPassReverse /proxy http://api.cloudcms.com:80

Now setup a Gitana account and add a simple test to the body section of your test html 

                <div id="field1">
                </div>
                <script type="text/javascript">
                $(function() {
                    new Gitana().authenticate('YOUR_USER_NAME','YOUR_PASSWORD').then(function() {
                        $("#field1").html('Hello '+this.authenticatedUsername+' Welcome to Gitana Cloud CMS!');
                    });
                });
                </script>

Save the html file and open it in a browser. You should get a welcome message.

Build Gitana SDK from source

All Alpaca source code is kept under Git revision control, which you can browse online.

If you have access to Git, you can connect to the repository here: 

git@github.com:sdk/sdk.git

Once you clone the SDK repository to your local machine and have Maven 3.0 or later installed, You can build SDK and run SDK examples using following Maven command 

mvn clean install jetty:run-exploded

Wait till Maven finishes the build process and brings up embedded Jetty server, you should now be able to check out all Alpaca examples from this URL 

http://localhost:18180/sdk/index.html
